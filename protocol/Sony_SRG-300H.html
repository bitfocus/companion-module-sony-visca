Sony SRG-300H Protocol Info

This document contains information about the VISCA Protocol for the following camera:

  • SRG-300H

Note: 1/2.8″ Exmor CMOS, 30× optical zoom, HDMI/DVI output. RS-232, RS-422, and VISCA over IP.

VISCA Command List

Part 1: Camera-Issued Messages

Function Packet Notes Comments
ACK y0 4z FF z: Socket Number Returned when the command is accepted.
Completion (commands) y0 5z FF z: Socket Number Returned when the command has been executed.
Completion (inquiries) y0 5z ... FF z: Socket Number Returned when the inquiry has been executed. Reply data follows byte 3.
Error – Syntax Error y0 60 02 FF Returned when the format is different or when a command with illegal parameters is accepted.
Error – Command Buffer Full y0 60 03 FF Could not accept a command that is received while two commands are currently being executed (two sockets have been used).
Error – Command Canceled y0 6z 04 FF z: Socket Number Returned when a command which is being executed in a socket specified by the cancel command is canceled. The completion message for the command is not returned.
Error – No Socket y0 6z 05 FF z: Socket Number Returned when no command is executed in a socket specified by the cancel command, or when an invalid socket number is specified.
Error – Command Not Executable y0 6z 41 FF z: Socket Number Returned when a command cannot be executed due to current conditions.
Network Change*1 y0 38 FF Issued when power is supplied to the camera.

*1 Not available for VISCA over IP.

Block Inquiries

Byte numbering is absolute: byte 0 is the address byte (y0), byte 1 is 50, data begins at byte 2, and the final byte is the FF terminator. Bit 0 is the least significant bit. The :: operator denotes concatenation, with the left operand as the most significant bits.
